# Requirements

## Must Have

### Authentication & Authorization
- Firebase Email, Google Login, Phone Login
- Hidden Admin Login (via normal login + role check)
- Strict "One Account = One Device" policy (Android ID / App UUID binding)
- Device change request & approval flow by Admin

### Admin Dashboard (In-App)
- Analytics Dashboard (Users, Videos, Revenue, etc.)
- User Management (Ban, Delete, Reset Device, Approve Device, Premium/Free, Search)
- Video Management (Add/Edit/Delete/Hide/Publish, Premium/Free, Categories, Google Drive Links)
- Categories Management

### Video Playback
- Google Drive stream URL extraction
- Modern Video Player (10s seek, subtitles, speed, quality, full-screen, auto-resume, gestures)
- Continue Watching (resume from exact timestamp)
- Watch History & Favorites

### Offline Downloads
- Download via Expo FileSystem
- AES-256 Encryption (saved in app sandbox)
- Offline playback support (on-the-fly decryption or chunk-based)
- Download management (Downloading, Paused, Completed, Delete)

### User Interface
- Premium Netflix-style Dark Theme UI
- Splash screen, Hero Carousel, Horizontal Rows, Glassmorphism, Skeletons/Shimmers
- Categories (Movies, Series, Courses, etc.)
- Instant Search (History, Recent, Popular, Suggestions)
- User Profile & Subscription Management

## Should Have
- Push Notifications via FCM (New movies, offers, updates)
- Scalability to 10k users before migrating from Google Drive to a CDN
- Subscription Tiers (Free, Premium, Lifetime, Monthly, Yearly)

## Could Have
- React Native Skia animations
- Root/Emulator detection
- Screenshot/recording detection

## Won't Have (Anti-features)
- Dedicated web-based Admin Panel
- Dedicated backend server (Node/Python) for video hosting
- Hollywood-grade DRM (Widevine/FairPlay)
- Video Transcoding pipelines

## Security & Architecture
- Firebase App Check
- Firestore Security Rules (Admins full access, users read-only published)
- Private Google Drive sharing strategy
- HTTPS Only
